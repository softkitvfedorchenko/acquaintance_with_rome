Rome

Sebastian McKenzie - создатель Yarn, Babel, член команды React Native, работал над “all-in-one” решением для JavaScript и TypeScript разработки. Rome - “all roads lead to Rome”

Зачем, почему, кто такой ?

Rome - линтер, компилятор, сборщик, "остальное" для JavaScript, TypeScript, JSON, HTML, Markdown, and CSS. ОРазрабатывается как единая замена Babel, ESLint, webpack, Prettier, Jest, другое.

Rome представляет собой с нуля реализацию полной цепочки инструментов javascript.

Rome написан на TypeScript и крутиться на Node.js.

Возможности Rome JSON:

Linter - выглядит как конфетка

Команды:

  • rome check
  • rome check --apply
  • rome check --review
  • rome check --format-only
  • rome check src/App.jsx
Проблемы:
  • нужно добавлять “type”: “module” в “package.json”
  • не парсит jsx из коробки, нужно использовать jsx, tsx расширение файлов. если меняем расширение файла js -> jsx сборщик не видит нужный файл
  • не распознает тесты
  • сырая документация

Достоинства:

  • удобное и информативное CLI

Linter в процессе

если, мы выберем do nothing -> просто перейдем к следующему пункту
